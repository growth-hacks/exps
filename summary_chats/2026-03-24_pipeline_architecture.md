## Часть 3 — Новая архитектура пайплайна

### Ключевые решения

1. **Эмоциональный дизайнер** — новый агент перед техническим дизайнером. Генерирует творческие концепции без ограничений компонентной библиотеки. Пользователь выбирает направление (Gate 3), потом обычный дизайнер реализует по гайдлайнам.

2. **CPO-reviewer** — стратегический ревью всего эксперимента (гипотезы + тексты + концепции). Выдаёт структурированный отчёт с указанием этапа проблемы. Не PASS/FAIL — аналитика для решения пользователя.

3. **Ревьюеры с петлями** — каждый агент работает в петле: FAIL → агент исправляет → ревьюер → снова. Лимит 3 итерации, потом эскалация.

4. **Оркестратор** — читает состояние папки эксперимента, сам решает что запускать следующим, останавливается на Gates.

5. **QA через Playwright** — реальный браузер, скриншоты, проверка GA4 и CDN.

